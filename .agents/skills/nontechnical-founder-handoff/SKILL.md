# Non-Technical Founder Handoff Skill

Use this skill whenever building UX, docs, setup instructions, or final responses.

## Principles

- Use plain English.
- Prefer buttons and guided flows over hidden configuration.
- Show what to do next.
- Explain errors with actions, not stack traces.
- Keep demo data available.
- Document environment variables clearly.
- Include screenshots or visual descriptions when possible.

## Founder-facing pages to maintain

- setup guide
- demo script
- launch checklist
- security checklist
- customer audit checklist

## Final response format to founder

When completing work, say:

1. What changed.
2. What they can click/test.
3. Which command/tests passed.
4. What to ask Codex next.
